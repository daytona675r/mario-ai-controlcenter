# Codex Handoff — Mario AI Control Center

## Why this file exists
This repository was started inside a ChatGPT project called "Job Switch Acceleration Campaign". Codex does not automatically know the full project context from that ChatGPT conversation, so this file packages the important background into the repo.

Read this file and `AGENTS.md` before making changes.

## The strategic goal
Mario wants to switch into an AI Infrastructure / MLOps / GenAI engineering role. The website should increase recruiter trust by making his experience easy to inspect.

The website should not be a vanity portfolio. It should behave like a searchable proof system.

## Target role family
- AI Infrastructure Engineer
- MLOps Engineer
- AI Platform Engineer
- Senior Software Engineer, AI Systems
- AI Solutions Engineer
- Forward-deployed style engineering roles, without overusing that title

## User profile summary
Mario is a senior software engineer with around 20+ years of professional experience. He has a strong background in enterprise software, SaaS, DevOps, CI/CD, platform engineering, and now production AI systems.

Current focus:
- MLOps / ML infrastructure
- CI/CD for PyTorch training pipelines
- MLflow
- Kubernetes
- Terraform
- AWS
- reproducible model training and promotion
- RAG and agent systems

Positioning to preserve:
> Senior engineer who builds production AI systems, not experimental ML demos.

## Site concept
Working name:
**AI Engineering Control Center**

The site should feel like:
- Stripe
- Linear
- Vercel
- Anthropic

It should not feel like:
- a game UI
- a Netflix clone
- a hacker terminal cliché
- a generic portfolio template

## Homepage priority
The homepage should make these visible:
1. Mario's positioning
2. searchable/filterable evidence
3. featured projects
4. recent insights/articles
5. skills backed by proof
6. capability matrix
7. timeline of evolution from enterprise software to production AI

Recent Insights should be more prominent than a small sidebar. It is a major differentiator because it shows thinking, communication, and technical judgment.

## Navigation philosophy
Keep recruiters inside the website first.

Project and article cards should open internal detail views/pages. GitHub and LinkedIn should be proof links from inside those pages, not the first destination.

## Recommended V1 behavior
- Search filters projects and insights
- Skill chips filter everything
- Recruiter Mode simplifies language
- Engineer Mode shows more technical details
- Project cards open detail view
- Insight cards open detail view
- External proof links are visible but secondary

## Content warning
For current employer work, keep domain/company details general. Share patterns and tooling, not private details.

Safe to mention:
- medical AI / computer vision generally
- MLflow, Kubernetes, Terraform, AWS, PyTorch, CI/CD
- model training pipelines
- validation gates
- reproducibility patterns

Avoid:
- internal URLs
- exact metric thresholds
- private datasets
- client names
- internal team details
- proprietary implementation details
